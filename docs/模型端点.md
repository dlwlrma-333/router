# 模型端点

本文档描述当前 Router 的模型端点分层与运行规则。

## 文档信息

- 最后核对日期：2026-05-05
- 关键代码路径：
  - `internal/admin/model/provider_model.go`
  - `internal/admin/model/channel_model.go`
  - `internal/admin/model/channel_model_endpoint.go`
  - `internal/admin/model/cache.go`
  - `internal/admin/controller/channel/model_runtime.go`
  - `internal/admin/controller/model.go`
  - `internal/relay/controller/text_bridge.go`

## 1. 核心结论

1. 供应商模型目录 `provider_models.supported_endpoints` 是渠道端点的默认候选来源。
2. 运行时端点真值只看 `channel_model_endpoints`。
3. `channel_tests` 只记录测试观察结果，不自动改写 `channel_model_endpoints`。
4. `/v1/models.supported_endpoints` 和 relay 选路读取同一套渠道端点真值。
5. 不再支持 `chat/completions -> responses/messages` 的隐式桥接；客户端应按 `/v1/models.supported_endpoints` 选择请求端点。

## 2. 分层职责

| 层级 | 主要载体 | 职责 | 是否运行时真值 |
| --- | --- | --- | --- |
| 供应商层 | `provider_models` | 维护模型归属、价格、官方/默认支持端点候选 | 否 |
| 渠道模型层 | `channel_models` | 维护渠道启用哪些模型、上游模型名、模型类型、价格覆盖 | 否 |
| 渠道端点层 | `channel_model_endpoints` | 维护 `channel + model + endpoint` 是否启用 | 是 |
| 测试层 | `channel_tests` | 记录每次模型端点测试结果、错误、耗时、是否流式 | 否 |
| 分组层 | `group_model_channels` | 维护分组可用的渠道模型组合 | 间接 |
| 用户接口层 | `/v1/models` | 暴露当前用户可达模型及 `supported_endpoints` | 只读 |

## 3. 供应商端点候选

`provider_models.supported_endpoints` 存储逗号分隔端点列表，例如：

- OpenAI text 默认候选：`/v1/responses`
- Anthropic text 默认候选：`/v1/messages`
- 其他 text 默认候选：`/v1/chat/completions`
- image/audio/video 按模型类型映射到对应原生端点

渠道详情的端点能力页从供应商候选生成可维护列表。候选只是可配置范围，不代表运行时已启用。

新增渠道流程里的模型测试端点也应优先来自供应商候选端点。测试通过只证明该端点在当时可用，不直接改变运行配置。

## 4. 渠道端点真值

`channel_model_endpoints` 是唯一运行时端点启用来源：

- relay 按请求路径过滤渠道时只读取该表。
- `/v1/models.supported_endpoints` 只汇总当前分组可达渠道里已启用的端点。
- 如果某个 `channel + model` 没有端点记录，则该渠道模型不会通过对应端点参与路由，也不会对外暴露该端点。

## 5. 模型测试

模型测试必须指定端点。测试结果写入 `channel_tests`，包含端点、是否流式、状态、耗时和错误信息。

测试结果不再自动修改 `channel_model_endpoints.enabled`。管理员需要根据测试结果明确启用或停用端点，避免“测试观察”和“运行配置”互相产生隐藏副作用。

新增渠道向导的最终确认页会基于已测试通过的端点写入 `channel_model_endpoints`。这是管理员确认后的配置保存动作，不属于测试任务副作用。

## 6. 文本端点协商

| 下游请求端点 | 上游要求 |
| --- | --- |
| `/v1/chat/completions` | 选中的渠道模型必须启用 `/v1/chat/completions` |
| `/v1/responses` | 选中的渠道模型必须启用 `/v1/responses` |
| `/v1/messages` | 选中的渠道模型必须启用 `/v1/messages`，且适配器支持 messages |

不满足时直接报错，不桥接、不降级、不按协议默认端点兜底。

## 7. 管理建议

1. 先维护供应商模型目录里的 `supported_endpoints`。
2. 渠道添加或编辑时，从供应商候选中确认该渠道模型实际启用的端点。
3. 对启用端点执行测试，把测试结果作为人工判断依据。
4. 联调第三方应用时，以 `/v1/models.supported_endpoints` 为准选择端点。
